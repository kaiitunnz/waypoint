## Bugs Report

- [x] **CORS default rejected Tailscale-IP origins.** Phones at `100.x.y.z:3000` hit `OPTIONS … 400` because the default origin allowlist was localhost-only. Fixed by defaulting `cors_allow_origin_regex` to cover localhost + Tailscale CGNAT.
- [x] **Stale token caused websocket reconnect storm.** `websocket.close(code=4401)` was called before `accept()`, so Starlette served HTTP 403 and the browser saw close code 1006 instead of 4401. Frontend never cleared the token. Fixed by accepting first, then closing with 4401.
- [x] **Codex agent messages split per word.** The adapter emitted one `agent_output` event per `item/agentMessage/delta` AND a duplicate on `item/completed`. Fixed by tagging delta events with `item_id`, suppressing the agentMessage completion event, and merging incoming `agent_output` events with the same `item_id` in the frontend (both initial history and live stream).
- [x] **Streamed tool output was split into multiple tool results.** Fixed by coalescing structured `tool_result` events on the frontend by `item_id`, appending streamed deltas into a single in-progress entry, and suppressing Codex `item/completed` tool-result duplicates when that tool already emitted streamed output. This keeps long-running command output such as `pytest` grouped under one result card.
- [x] **Backend termination stuck (codex side).** `terminate_session` issued `turn_interrupt` while the streaming task held `state.transport_lock` parked in an uncancellable `to_thread(next_notification)`, so shutdown deadlocked until a second Ctrl+C force-killed uvicorn. Fixed by closing the codex client first (drops EOF on stdio, releases the lock and lets the streaming task observe its cancellation) and dropping the now-redundant `turn_interrupt` from the termination path.
- [x] **Backend termination stuck (websocket side).** Even after the codex fix, an active websocket held uvicorn through Ctrl+C. `await queue.get()` raised `CancelledError` on shutdown, which our handler didn't catch; Starlette's exception machinery then tried to send a 1011 close frame on the tearing-down transport and that send hung. Fixed by catching `CancelledError` alongside `WebSocketDisconnect` in both ws handlers, sending our own best-effort close in `finally`, and capping uvicorn's `timeout_graceful_shutdown` to 5s so any future regression can't hold the process indefinitely. Single Ctrl+C now exits in ~1s with an active ws.
- [x] **Failed to fetch when working directory is ~.** The frontend sent the literal `~/` and the backend passed it straight to `subprocess.Popen(cwd=...)`, which doesn't expand tilde. For local launches `runtime.create_session` now resolves `~` via `Path.expanduser()` before storing or launching. For SSH launches the remote path was also being single-quoted via `shlex.quote`, suppressing the remote shell's tilde expansion; `_quote_remote_path` now leaves the `~`-prefix bare and quotes only the suffix so `cd ~/foo` reaches the remote shell intact.
- [x] **Mobile webpage has weird aspect ratio.** Fixed by switching the app shell away from `100vw` sizing, adding an explicit mobile viewport declaration, and hardening overflow wrapping so long URLs / transcript content no longer push the page wider on phones.
- [x] **Auto-scroll does not work well.** Fixed by making the session view auto-scroll on streamed transcript updates as well as appended events, using the full `events` array rather than only `events.length`.
- [x] **Performance issues.** Fixed by batching streamed websocket events into one React update per animation frame, rendering the transcript from a deferred event view, and memoizing transcript cards so unchanged rows do not re-render on every tool-result delta.
- [x] **Slow response when typing into the reply textbox.** Fixed by moving the draft state into a memoized reply composer so typing no longer re-renders the full transcript on each keystroke.
- [x] **Blank whitespace at the top and bottom of the page.** Fixed by splitting the html/body CSS so the rubber-band area paints `var(--bg-deep)` (and added a matching `themeColor` viewport tag) instead of the browser's default white. The earlier attempt put `overscroll-behavior` on both elements together with `overflow-x: hidden` and broke trackpad scroll on desktop because wheel events landed on whichever of html/body the browser didn't pick as the scroll root; html now only carries the dark background and body owns the scroll behaviour.
- [x] **Phone notch eats into the page content.** Fixed by extending `.page-shell` padding with `env(safe-area-inset-*)` for both desktop and the mobile media query. The PWA already runs with `viewportFit: "cover"` and a black-translucent status bar, so the screen draws edge-to-edge under the notch / home-indicator; the safe-area insets keep the app bar and bottom controls clear of those cutouts on iPhone X+ devices in standalone PWA mode.
- [x] **Auto-scroll is janky.** Replaced the ad-hoc `scrollIntoView` against a sentinel placed above the reply composer with a `ResizeObserver` on the page section that triggers `window.scrollTo({ top: scrollHeight })` whenever transcript content reflows (markdown layout, streamed deltas, image loads, approval card appearing). Auto-scroll now reaches the actual page bottom — composer included — and reliably re-anchors as content grows, instead of stopping short above the composer or skipping a deferred render.
- [x] **Frequent need to relogin.** Tokens now slide their expiry forward inside `TokenStore.validate` whenever more than half the configured TTL has elapsed, so any active use refreshes the window. Default TTL is now 30 days (was 7) to better match the cadence of a personal tool, and to outlast the iOS PWA storage-eviction window for users who don't open the app every week.
- [x] **Approval on top.** The structured `ApprovalCard` now renders just before the `ReplyComposer` instead of above the view-toggle, so the Approve / Decline buttons sit next to the textarea where the thumb already is on phones.
- [x] **The session schedule list should be clearable.** `DELETE /api/schedules/{id}` now hard-deletes terminal entries (launched / cancelled / failed) while still soft-cancelling pending ones, and a new `POST /api/schedules/clear-history` bulk-removes every non-pending row. The Schedule panel's "Recent" section gains a `Clear history` button and per-row entries show `Dismiss` instead of having no action once the schedule has finished.
- [x] **SQLite error on backend.** `sqlite3.InterfaceError: bad parameter or other API misuse` raised from `Storage.get_token_expiry` under load. Root cause: the singleton `sqlite3.Connection` was opened with `check_same_thread=False` but had no user-side serialization, so concurrent FastAPI threadpool requests (auth check is the hot path) interleaved cursor calls on the same connection and corrupted its state machine. Fixed by adding a `threading.RLock` on `Storage` and a `_synchronized` decorator applied to every public method that touches the connection; RLock so reentrant callers like `update_session` (which calls `get_session`) don't self-deadlock. Regression test in `test_storage_serializes_concurrent_threads` hammers `get_token_expiry`/`refresh_token_expiry` from a 16-thread pool against 32 tokens.

- [x] **Support /compact.** Codex now routes `/compact` through the SDK's `client.thread_compact(thread_id)` (drained in `_stream_compact` until `thread/compacted` arrives), with an active-turn rejection. Claude Code's CLI advertises `compact` as a built-in slash command in `--input-format=stream-json`, so the runtime forwards `/compact` unchanged and the adapter renders the resulting `system/status` (`compacting`, `compact_result=…`) and `system/compact_boundary` events as transcript-friendly notes (e.g. `Context compacted (manual) · 27000 → 1200 tokens · 23000 ms`). The Important transcript filter now matches `compact` so users see the lifecycle without flipping to All events.
- [x] **Auto-complete for built-in commands.** Reply composer now opens a suggestion dropdown when the draft starts with `/` and contains no whitespace yet, listing the four backend-intercepted commands (`/help`, `/status`, `/permissions`, `/compact`) with descriptions. Keyboard nav: Tab or plain Enter completes (preserved Cmd+Enter for send), ↑/↓ moves selection, Escape dismisses. Gated to structured transports (codex_app_server / claude_cli) since tmux skips `_handle_builtin_command`.
- [x] **Support session pagination and pinning.** Sessions table gains a nullable `pinned_at` column (migrated via `_ensure_column`); runtime exposes `set_pinned` plus `POST/DELETE /api/sessions/{id}/pin` endpoints, broadcasting the usual `session_list_update` so all clients re-render. Frontend home page splits the list into a top "Pinned" section (sorted by most-recently-pinned) and a "Recent" section paginated at 10 per page; per-row `☆ Pin / ★ Pinned` link-button on every card (mobile-friendly tap target — context menus are unreliable on mobile because long-press collides with iOS selection / share sheet, so an explicit button beats `oncontextmenu`).
- [x] **Revise built-in commands to forward to backends.** Removed Waypoint's `_handle_builtin_command` synthesizing for `/help`, `/status`, `/permissions`. They now flow straight to the underlying transport: Claude CLI's stream-json mode parses slash commands itself and emits its own `system/status` events (the same path `/compact` already used), and Codex passes them to the agent. The single remaining Waypoint-side route is Codex `/compact` → `thread_compact()` SDK RPC, since the Codex app-server doesn't parse user text as a control command. Helper renderers `_format_builtin_status` / `_format_builtin_permissions` are gone; runtime tests now assert forwarding instead of interception.

  **Out of scope here** — `/fork` (Codex SDK exposes `thread_fork` but that's a backend feature, not a Waypoint slash command) and `/plan` (Codex's `ModeKind` plan/default is initial-only at `thread_start`; no mid-session toggle exists; Claude has no mid-session mode switching at all). Both belong to the mode-switching feature, which has its own constraints.

With `--yes`, removes them and recreates the empty `data_dir` / `sessions_dir`. Config is never touched. Tests in `tests/test_cli_reset.py` cover dry-run, confirmed wipe, and the no-op path.

It mirrors the Latest pattern: `position: sticky; top: 12px;` so it floats at the viewport top while the row scrolls past, conditional render once `window.scrollY > 320`, suppressed in terminal view.
